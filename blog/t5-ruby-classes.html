Introduction to Ruby Classes

Creating Classes and Objects

May 10, 2015

Classes allow you to create a basic template for a set of similar objects. For example, every time you create a new object of the "Car class" it would be able to do methods like "honk", "drive_forward" and "rev_engine". It would also have instance variables like @make, @model and @color. These are things all cars have in common (they may have different makes, but they all at least have). Every time you create a new Car object, it will "come with" these characteristics from the moment of initialization. If you're going to be creating a lot of "cars", this will save you a good deal of time and promote consistency among instances of the Car class.

I'd like to play around with our Car class example, and introduce you to my car, "Suby" the green Subaru Outback...

class Car
    def initialize(make,model,color,mpg)
        @make = make
        @model = model
        @color = color
        @mpg = mpg
    end
end

Suby = Car.new('Subaru','Outback','green',27)

In the above code I first creating the Car class. I then go on to define the initialization method (used to create new instances of the Car class). The initialization method requires four parameters: make, model, color and MPG. Below this, after I've closed the class, you can see that I'm creating a new instance of the Car class called Suby. For the initialization parameters, I pass use the values of 'Subaru', 'Outback', 'green' and 27. We can create a second car just as easily.

Simon = Car.new('Honda','Civic','blue',34)

class Car
    def basic_info
        puts "This is a #{@color} #{@make} #{@model} that gets #{@mpg} MPG."
    end
end

Suby.basic_info
Simon.basic_info

You can see above that we added "Simon", a new instance of the Car class. After entering Simon's information, we went on to define a new method for the Car class, basic_info. This method displays a sentence with information about the car. Below this, we then called this basic_info method on both Suby and Simon. This results in the below output.

This is a green Subaru Outback that gets 27 MPG.
This is a blue Honda Civic that gets 34 MPG.

You can see that both Suby and Simon have their own values for @make, @model, @color and @mpg. This is because these are instance variables. Each instance of the Car class (Suby and Simon are each an instance) has their own value for each instance variables. Instance variables are easy to spot because they have an '@' in front of the variable name.

This is a very basic example of how to use classes in Ruby, but I hope it was helpful. There is a lot more to discuss, so hopefully I'll be able to address it in a future blog post.
