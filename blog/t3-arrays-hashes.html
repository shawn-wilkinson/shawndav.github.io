Organizing Information in Ruby

An Introduction to Arrays and Hashes

April 26, 2015

Arrays and Hashes are unique because they can actually contain other objects. They are useful for saving and organizing informaiton, and prove essential to programming in Ruby. Today I'm going give a very introductory overview of arrays, hashes and examples of how they can be used.

Arrays: Arrays can contain a group of objects in an ordered list. Each object in the arrays has a number, which makes arrays useful for maintaining order. For example, an array would be useful for keeping both a shopping list (where order doesn't matter) and names of people waiting for a table at a busy restaurant (where order matters).

shopping_list = ["eggs","milk","coffee","ice cream","dog food"]

waiting_list = ["Smith","Wilson","Cooper"]

We can play with this a bit more. Instead of text, let's put an array inside of the our array waiting_list. For each of the families listed, we're going to include their last name, the number of people in their party and also a true / false to indicate if it's a special occasion.

waiting_list = [
["Smith", 4, true],
["Wilson", 2, false],
["Cooper", 10, true]
]

As you can see within the main array, there are three different arrays. The Smith family is the first array. We can see that they have a party of four and they are celebrating something tonight. Arrays can get much more complex than this, but this gives you an idea as to useful and flexible they can be.

Hashes: Hashes are simmilar to Arrays in that they also contain other objects. While arrays contain an ordered list of individual objects, Hashes contain multiple pairs of "keys" and "values". The keys can help provide context for the information in value. For example, let's say you wanted to store information on all of your friend's favorite things. We could create a hash called courtneys_favorites and inside have "keys" (favorite color, favorite food, favorite movie) that correspond to "values" (blue, burritos, Reno 911).

courtneys_favorites = {
"favorite color" => "blue",
"favorite food" => "burritos",
"favorite movie" => "Reno 911"
}

In the code above, the key favorite color points to the value blue. This is very simple example of hashes can be useful for saving and providing context for information.

Mix It Up: Let's put hashes inside an array. Let's use the friend example, we're going to have an array of three of our friends. Each item in that array is going to be a hash. Inside each hash we are going to use keys and values to store each friend's name, their favorite food and their hometown.

friends = [
{"name" => "Stephanie", "favorite food" => "fried pickles", "hometown" => "Atlanta"},
{"name" => "Nicole", "favorite food" => "ribbon candy", "hometown" => "Rochester"},
{"name" => "Leah", "favorite food" => "gyoza", "hometown" => "Seattle"}
]

As you can see, we have information about three friends stored in our friends array. Each friend has its own hash within the array. Very interesting stuff!

You can see from these examples how useful both arrays and hashes can be, and how they can work together to organize information.
